# InputDialog のテスト

## 何をテストするか

`InputDialog` はブラウザの `window.prompt` の代替となるモーダルコンポーネント。
テキスト入力、送信/キャンセルのコールバック、IME 対応、入力検証を検証する。

## なぜテストするか

branch 名入力など、ユーザーにテキスト入力を求めるフローをテスト可能にするため。
`window.prompt` は jsdom では戻り値を制御できず、UI としても一貫性がない。

## テストケース

### 表示
- メッセージがラベルとして表示されること
- 入力フィールドに自動フォーカスされること
- 初期値が設定できること

### 操作
- OK ボタンクリックで onSubmit が入力値とともに呼ばれること
- Enter キーで onSubmit が呼ばれること
- キャンセルボタンクリックで onCancel が呼ばれること
- Escape キーで onCancel が呼ばれること
- 背景クリックで onCancel が呼ばれること

### 入力検証
- 空文字列では OK ボタンが無効になること
- 空白のみの入力では OK ボタンが無効になること

### IME 対応
- IME 変換中の Enter では onSubmit が呼ばれないこと
- IME 確定後の Enter では onSubmit が呼ばれること
